Git Commands for Beginners

GIT :

Git is a distributed version control system (VCS) for tracking changes in any set of files, originally designed for coordinating work amoung programmers coorperating on source code during software development.It Goals include speed, data integrity and support for distributed, non-linear workflows

GITHUB :

GITHUB is a subsidary of Microsoft which provides hosting for software development and version control using git. It offers the distributed version control and source code management functionality of git. It lets the people work on files and easily merge their changes in with the master/main branch of the project

Installation :

  • Navigate to git-scm

  • Download latest git version to your local machine

  • Install git

Commands :

open command prompt (cmd) / git bash

git --version It gives the current installed version which is in our local machine
git config --global user.name “your user name” sets your username to git in your local machine
git config --global user.email “your mail id” will set your email to git in your local machine
git init initializes git in local folder, it creates .git folder in which you have initialized this command
git status represents the file status i.e; untracked, staging and commit
git add "filename" add the file in the working directory to staging area
git add . add all the files in the working directory to staging area
git commit -m “commit message” This command will commit all the files which are in staging area
git commit --amend -m "commit message" used to change commit message
git log it will give the commit log in chronological order
git log --oneline describes the commit messages in a short manner
git log --until = “Date” generates commit message from starting to the given date
git rev-list HEAD --count returns the count of commit messages
git checkout "7characters sha key"> jumps to the respective commit
git checkout "main/master" get you to the main/master branch of project.
git checkout "branch name" takes you to another branch which you have given the branch name
git remote we can check the remotes available in the folder or repository
git remote add "remote name" "url" adds new remote
git remote -v displays the current url which it holds
git push "remote name" "branch" pushes all your local files to the github repository(if you are using github as vcs).
git pull "remote name" "branch name" will synchronize the local repository with git repository
git fetch "remote" Fetch all of the branches from the repository. This also downloads all of the required commits and files from the other repository.
git fetch "remote name" "branch name" Same as the above command, but only fetch the specified branch.
git branch throws the number of branches available in our local repository
git branch "new branch name" creates the new branch
git merge "branch name" used to merge current branch to master/main
git reset --hard HEAD^ deletes latest commit
git reset --hard HEAD~"number" deletes number of commits form counting from head
git rebase --onto "branch name"~"number1" "branch name"~"number2" "branch name" deletes specific commit
git branch -d "branch name" deletes the brach in local after merging
git branch -D "branch name" deletes the branch in local without merge
git push "remote name" --delete "branch name" deletes branch in remote
git show used to view expanded details on Git objects such as blobs, trees, tags, and commits
git clean -n to see dry run
git clean -f force to remov untracked files
